Jenkins持续集成(1)

2018-05-07
Jenkins, 持续集成

最近开始接触持续集成的相关内容，这一系列作为自己的一个学习整理，会不断充实下去。这篇笔记主要记录一下自己在搭建Jenkins时过程和遇到的一些问题，成功运行本地的API脚本。当然还有好多坑。。。

工具

此次搭建，使用了Tomcat作为容器。

  • jenkins 2.119
  • tomcat-8.5.24
  • Mac 10.13.2

搭建步骤

1、首先从官网下载一个Tomcat，不需要改动。本地可以启动运行就行。
2、如果直接在Mac上安装，下载.pkg格式的即可，但若要放在tomcat上，就需要下载.war格式的jenkins。下载地址
3、下载后将.war放在tomcat下wepapps文件夹下即可
4、重启tomcat，浏览器输入地址http://localhost:18080/jenkins/ 会进入到jenkins的初始化页面。默认端口号为8080，18080是我后来自己改的。

需要输入自己的密码，初始密码存储的位置：/Users/用户名/.jenkins/secrets/initialAdminPassword，这个初始密码文件会在创建用户后消失.
之后需要选择要安装的插件，可根据自己的需要进行安装，后期需要还可以在jenkins里安装，安装的过程需要十几到几十分钟，等着就行
之后新建用户，输入用户名、密码，登录，OK

执行本地API脚本

首先需要确保自己已经安装了RF的插件，我使用的是Advanced这个tab下上传插件的方式。RF插件下载地址 下载下来是一个.hpi文件。上传之后就可以使用。
之后新建一个job，填入自己的job名称，这里选择自由风格的项目

Job的配置页面

运行本地API脚本比较简单，source code management中选择None就可以，Build里选择execute shell，在shell输入框里写入本地执行RF脚本的执行命令

Build Trigger中可以配置该job的定时执行。定时执行配置语法挖坑待填。
